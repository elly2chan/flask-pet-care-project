API Documentation

Project Overview

This API provides a platform for managing pets, products, orders, and users. The system is designed to allow users to register, view products, place orders, and manage pets, with role-based access control for administrative functionalities.

Models

UserModel

  • id: Integer, primary key
  • email: String, unique
  • password: String
  • first_name: String
  • last_name: String
  • phone: String
  • role: Enum (user/admin)

PetModel

  • id: Integer, primary key
  • name: String
  • gender: Enum (male/female)
  • date_of_birth: Date
  • breed: String
  • owner_id: Integer, foreign key (User)
  • pet_type: Enum (dog/cat)
  • is_stray: Boolean
  • microchip: Boolean

OrderModel

  • id: Integer, primary key
  • created_on: DateTime
  • address: String
  • customer_id: Integer, foreign key (User)
  • product_id: Integer, foreign key (Product)
  • product_title: String
  • status: Enum (pending/approved/rejected)

Managers

AuthManager

Handles authentication tasks such as login, registration, and token generation.

OrderManager

Manages orders: placing new orders, approving or rejecting them, and updating order statuses.

ProductManager

Handles product-related tasks like adding, editing, and deleting products.

Endpoints

Public Endpoints

  • POST /register - Register a new user
    Request Body: 
                {
              "email": "user@example.com",
              "password": "strongpassword",
              "first_name": "John",
              "last_name": "Doe"
            }
    Response: 
                {
              "message": "User registered successfully"
            }
  • POST /login - Log in a user
    Request Body: 
                {
              "email": "user@example.com",
              "password": "strongpassword"
            }
    Response: 
                {
              "token": "jwt_token_here"
            }
  • GET /about - Get information about the API
    Response: 
                {
              "message": "This is the about page for the API."
            }

Restricted Endpoints

  • POST /pets/add_pet - Add a new pet (Authenticated users only)
    Request Body: 
                {
              "name": "Fluffy",
              "gender": "female",
              "breed": "Golden Retriever",
              "pet_type": "dog"
            }
    Response: 
                {
              "message": "Pet added successfully"
            }
  • POST /orders/place_order - Place an order (Authenticated users only)
    Request Body: 
                {
              "address": "123 Main St",
              "product_id": 1
            }
    Response: 
                {
              "message": "Order placed successfully"
            }
  • PUT /orders/approve_order/{order_id} - Approve an order (Admin only)
    Response: 
                {
              "message": "Order approved"
            }
